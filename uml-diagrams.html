Sequence Diagram — TLS Scan Flow

End-to-end scan flow: from user initiation through TLS probing, CBOM generation, quantum assessment, to report delivery.

sequenceDiagram
    actor Analyst as Security Analyst
    participant UI as React Frontend
    participant API as Express API
    participant Scanner as TLS Scanner Engine
    participant Target as Target Endpoint
    participant Assessor as Quantum Assessor
    participant CBOM as CBOM Generator
    participant DB as MongoDB
    participant Email as Email Service
    participant Chatbot as AI Chatbot (RAG)
    participant VectorDB as Vector KB

    Analyst->>UI: Enter target URL & config
    UI->>API: POST /api/scan {targets, config}
    API->>DB: Create scan record (status: running)
    API-->>UI: scanId + status: queued

    loop For each target endpoint
        API->>Scanner: Probe target
        Scanner->>Target: TLS ClientHello (v1.0-1.3)
        Target-->>Scanner: ServerHello + Certificate Chain
        Scanner->>Target: Enumerate cipher suites
        Target-->>Scanner: Supported cipher suites
        Scanner-->>API: Raw TLS data
    end

    API->>Assessor: Classify all crypto parameters
    Assessor->>Assessor: Check NIST PQC OIDs
    Assessor->>Assessor: Compute quantum score (0-1000)
    Assessor-->>API: Labels + Recommendations

    API->>CBOM: Generate CBOM records
    CBOM->>CBOM: Compute SHA-256 integrity hash
    CBOM-->>API: CBOM JSON + integrity hash

    API->>DB: Store CbomRecords + scan results
    API->>DB: Log audit entry
    API-->>UI: Scan complete notification

    UI-->>Analyst: Display dashboard + results

    opt Scheduled Email Report
        API->>CBOM: Generate PDF report
        CBOM-->>API: PDF buffer
        API->>Email: Send to authorized Gmail recipients
        Email-->>API: Delivery confirmation
    end

    opt AI Chatbot Query
        Analyst->>UI: Ask question about results
        UI->>API: POST /api/chatbot {query}
        API->>Chatbot: Process query
        Chatbot->>VectorDB: Retrieve relevant PQC docs
        VectorDB-->>Chatbot: Context chunks
        Chatbot->>Chatbot: RAG inference
        Chatbot-->>API: Generated answer
        API-->>UI: Chatbot response
        UI-->>Analyst: Display answer
    end

State Diagram — Scan Lifecycle

States a scan transitions through from creation to completion or failure.

stateDiagram-v2
    [*] --> Queued : Scan initiated
    Queued --> Validating : Validate targets
    Validating --> Invalid : Validation failed
    Invalid --> [*]

    Validating --> Running : Targets valid
    Running --> Probing : For each target
    Probing --> HandshakeSuccess : TLS handshake OK
    Probing --> HandshakeFailed : Connection failed
    HandshakeFailed --> Retrying : Retry (max 3)
    Retrying --> Probing : Retry attempt
    Retrying --> TargetFailed : Retries exhausted
    TargetFailed --> Running : Continue next target

    HandshakeSuccess --> Extracting : Parse certs + ciphers
    Extracting --> Assessing : Quantum classification
    Assessing --> Labeling : Compute score + label
    Labeling --> Running : Store CBOM + next target

    Running --> Completing : All targets done
    Completing --> GeneratingReport : Build CBOM report
    GeneratingReport --> HashingIntegrity : SHA-256 hash
    HashingIntegrity --> Completed : Save to DB

    Completed --> [*]

    Running --> Failed : Unrecoverable error
    Failed --> [*]

    note right of Assessing
        Uses NIST FIPS 203/204/205/206
        OID matching for PQC detection
    end note

    note right of Completed
        Triggers: Dashboard refresh,
        Audit log, Optional email report
    end note
